# Add 2v2 Multiplayer with Go Backend

## Overview
Add online multiplayer to the existing Hokm card game. Go backend runs the game engine server-authoritatively. Players create/join rooms via room codes. Remaining seats (2-4 humans) are filled with server-side bots. Existing local mode stays untouched.

## Architecture

```
Frontend (React) <--WebSocket--> Go Backend (game server)
                                    |
                                    +-- Game Engine (mirrors TS logic)
                                    +-- Bot AI (mirrors TS bot logic)
                                    +-- Room Manager
```

## Part 1: Go Backend

### Project Structure
```
server/
  main.go              -- entry point, HTTP/WS server
  go.mod
  room.go              -- room lifecycle (create, join, start, end)
  game.go              -- game engine (port of TS engine)
  bot.go               -- bot AI (port of TS bot)
  player.go            -- player connection management
  messages.go          -- all message types (JSON structs)
```

### Key Types (`messages.go`)

```go
// Client -> Server
type ClientMessage struct {
    Type    string          `json:"type"`
    Payload json.RawMessage `json:"payload"`
}

// Message types from client:
// "create_room"   -> { playerName: string }
// "join_room"     -> { playerName: string, roomCode: string }
// "choose_hokm"   -> { suit: string }
// "play_card"     -> { card: { suit: string, rank: string } }
// "start_game"    -> {} (room creator signals ready)

// Server -> Client
type ServerMessage struct {
    Type    string      `json:"type"`
    Payload interface{} `json:"payload"`
}

// Message types from server:
// "room_created"   -> { roomCode: string, playerId: string }
// "room_joined"    -> { roomCode: string, playerId: string }
// "player_joined"  -> { players: []PlayerInfo }
// "game_state"     -> { full GameState (sanitized for viewer) }
// "game_started"   -> {}
// "error"          -> { message: string }
```

### Game Engine (`game.go`)
- Port the TS engine logic to Go: `createDeck`, `shuffleDeck`, `deal`, `sortHand`, `canBeat`, `pickWinner`
- `GameState` struct mirrors `LocalGameState` from TS
- All state transitions are methods on a `Game` struct
- Server runs the full game loop: deal -> choose hokm -> play tricks -> resolve -> next round -> match end
- **Server sends sanitized state to each player** (hide other players' hands, only reveal own hand + played cards)

### Bot AI (`bot.go`)
- Port the TS bot logic: `botChooseHokm`, `botPlayCard`, `leadCard`, `followSuit`, `trumpCard`, `throwOff`
- Bots run server-side with a 800ms delay (matching frontend expectations)

### Room Manager (`room.go`)
- In-memory map of rooms: `map[string]*Room`
- Room states: `Waiting` (1+ players joined) -> `Playing` -> `Finished`
- Room code: 4-character alphanumeric (e.g., "A3K9")
- When game starts: fill empty seats (max 4) with bots
- Auto-cleanup rooms after 30min inactivity

### Game State Flow
```
1. Player A creates room -> gets roomCode, waits
2. Player B joins room -> sees Player A in lobby
3. Either player clicks "Start" -> server fills remaining seats with bots
4. Server deals cards, picks random hokmPlayer
5. Server sends initial state (5 cards each) to all human players
6. Server runs game loop:
   a. If hokmPlayer is bot -> bot chooses hokm (800ms delay)
   b. If hokmPlayer is human -> wait for choose_hokm message
   c. Deal remaining cards (8 more per player)
   d. For each trick:
      - If current turn is bot -> bot plays (800ms delay), broadcast
      - If current turn is human -> wait for play_card message
      - After 4 cards -> resolve trick (1500ms delay), broadcast
   e. After 13 tricks -> update scores, start new round or end match
7. Broadcast full game state after every action
```

### Sanitized State
Each player receives a `GameState` where:
- Their own hand is included
- Other players' hands are replaced with `cardCount: N` (no card details)
- Everything else (trick, scores, phase, turn) is shared

### Main Server (`main.go`)
- HTTP server on port 8080
- WebSocket upgrade on `/ws`
- Health check on `/health`
- CORS enabled for development

### Online Game Hook (`useOnlineGame.ts`)
```
State:
  - roomCode: string | null
  - playerId: string | null
  - phase: 'idle' | 'lobby' | 'playing' | 'finished'
  - game: OnlineGameState | null
  - players: PlayerInfo[] (lobby player list)

Functions:
  - createRoom(playerName) -> connects WS, sends create_room
  - joinRoom(playerName, roomCode) -> sends join_room
  - startGame() -> sends start_game (creator only)
  - chooseHokm(suit) -> sends choose_hokm
  - playCard(card) -> sends play_card

Effects:
  - Listen for server messages, update state accordingly
  - After each game_state, trigger UI update (no local bot timers needed)
```

### OnlineGameState Type
```ts
interface OnlinePlayerInfo {
  id: string
  name: string
  position: PlayerPosition
  cardCount: number  // only known info about other players
  tricksWon: number
}

interface OnlineGameState {
  id: string
  phase: TrickPhase
  myPosition: PlayerPosition
  myHand: Card[]
  players: OnlinePlayerInfo[]
  hokmSuit?: Suit
  currentTrick: Trick
  northSouthScore: number
  eastWestScore: number
  turn: PlayerPosition
  roundNumber: number
  matchWinner?: 'ns' | 'ew'
}
```
